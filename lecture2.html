1什么是 MCP？——给 AI 装上"万能接口" ⏱ ~40分钟

MCP（Model Context Protocol，模型上下文协议） 是 Anthropic 于 2024 年底发布的开源协议。它的核心使命是：让 AI 模型能够安全、标准化地与外部工具和数据源通信。

一个比喻理解 MCP

如果把 AI 比作一个人的大脑，那么 MCP 就是这个人身上的神经系统——大脑通过神经控制双手（工具）、感知外界（数据）、协调行动（工作流）。没有 MCP 的 AI 就像"缸中之脑"：能思考，但无法与外界互动。

统一标准

如同 USB 协议让任何设备都能即插即用，MCP 让任何工具都能被 AI 调用——一次开发，到处使用

能力扩展

通过 MCP Server 为 AI 添加：数据库查询、文件操作、API 调用、本地程序执行等能力

客户端-服务器架构

Client（Trae/Cursor等 AI IDE）← MCP 协议 → Server（你写的工具服务），双方通过 JSON-RPC 通信

安全可控

用户授权机制——MCP Server 只能在用户明确允许的范围内操作，不会越权访问文件或网络

生态开放

任何人都可以开发并分享 MCP Server，社区已有数百个现成的 Server 可直接使用

教学意义

教师可以为 AI 接入学科数据库、成绩系统、课件库，让通用 AI 变成专属教学助手

2MCP 的通用结构——学会"搭骨架" ⏱ ~20分钟

无论你教哪门学科，MCP Server 的结构都是一模一样的。掌握这个通用模板，你就能为任何学科搭建工具：

# ═══════════════ MCP Server 通用模板 ═══════════════
# 以下结构适用于任意学科——只需替换 "## 你的学科内容" 部分

from mcp.server import Server
from mcp.server.models import InitializationCapabilities

# ① 创建服务器——给它起个名字
server = Server("my-teaching-tools")

# ② 准备你的学科数据或工具
## ============ 这里替换为你学科的内容 ============
## 语文：课文数据库、诗词库、字词释义字典
## 数学：公式库、题库、几何图形参数
## 英语：单词库、语法规则、例句集合
## 思政：法律条文、时事案例库、价值观讨论题
## 音乐：乐理知识库、曲谱数据、音乐家信息
## 信息：代码模板库、算法动画数据、API文档
## ===============================================

# ③ 定义工具——每个 @server.tool() 就是 AI 能调用的一个功能
@server.tool()
async def my_tool(param: str) -> str:
    """工具描述——AI 会根据这个描述决定何时调用它"""
    # 在这里实现你的逻辑
    return "结果"

# ④ 启动服务器
if __name__ == "__main__":
    server.run()
关键认知：80% 的 MCP 代码是"骨架"（每个学科都一样），只有 20% 是你的"学科专属内容"。这意味着你只需学一次，就能复用终身。

3六大学科 MCP 案例——让 AI 做到它原本做不到的事 ⏱ ~60分钟

关键认知：MCP 不是知识库，是"手"和"眼"
AI 的训练数据里已经有古诗词、数学公式、语法规则——这些不需要 MCP。MCP 的真正价值在于：读取本地文件、执行代码计算、操作数据库、生成真实文件、调用外部系统。下面六个案例，每个都让 AI 做一件它"想得到但做不到"的事。
语文：作文批改系统 MCP

核心价值：读取本地作文文件 → 逐篇批改 → 导出批改报告（Word/Excel）——AI 自己只能分析你粘贴给它的文字，有了 MCP 它就能直接操作你电脑上的文件。

# chinese_essay_mcp/server.py
import os, json
from pathlib import Path
from docx import Document  # 操作 Word 文件

server = Server("essay-grader")

@server.tool()
async def read_essays_from_folder(folder_path: str) -> dict:
    """读取文件夹中所有学生作文（.txt / .docx），AI 自己做不到这一点"""
    essays = {}
    for file in Path(folder_path).glob("*.txt"):
        essays[file.name] = file.read_text(encoding="utf-8")
    for file in Path(folder_path).glob("*.docx"):
        doc = Document(file)
        essays[file.name] = "\n".join(p.text for p in doc.paragraphs)
    return {"文件数": len(essays), "作文列表": essays}

@server.tool()
async def batch_grade_essays(folder_path: str, criteria: dict) -> dict:
    """批量批改——读取→分析→打分→生成报告，一键完成"""
    results = {}
    for file in Path(folder_path).glob("*.txt"):
        text = file.read_text(encoding="utf-8")
        # 由 AI 逐篇分析：结构、语言、内容、卷面
        results[file.stem] = {
            "内容分": "由AI根据criteria判断",
            "语言分": "由AI根据criteria判断",
            "总评": "由AI生成个性化评语"
        }
    return results

@server.tool()
async def export_grading_report(results: dict, output_path: str) -> str:
    """将批改结果导出为 Word 报告——真文件，可直接打印发给学生"""
    doc = Document()
    doc.add_heading("作文批改报告", level=1)
    for name, result in results.items():
        doc.add_heading(name, level=2)
        for key, val in result.items():
            doc.add_paragraph(f"{key}：{val}")
    doc.save(output_path)
    return f"报告已保存至 {output_path}，可直接打开查看"

没有 MCP 时：老师手动复制每篇作文粘贴给 AI → AI 逐一分析 → 老师手动整理结果到 Excel。
有了 MCP 后：老师说"批改 D:\作文\期中 文件夹下的 40 篇作文"→ AI 自己读文件、逐篇批改、生成 Word 报告。

数学：动态出题与批改 MCP

核心价值：用 Python 精确计算出题（不是 AI 猜答案）、逐题批改全班作业、自动生成错题统计——AI 自己算数学题经常算错，MCP 用代码算，100% 准确。

# math_exam_mcp/server.py
import random, json
from pathlib import Path

server = Server("math-exam-generator")

@server.tool()
async def generate_math_paper(grade: int, topics: list, count: int = 10) -> dict:
    """出题 + 答案一起生成——答案由 Python 精确计算，不是 AI 猜的"""
    paper = {"年级": f"{grade}年级", "题目": []}
    for i in range(count):
        a, b = random.randint(1, 100), random.randint(1, 100)
        if "四则运算" in topics:
            paper["题目"].append({
                "题号": i+1,
                "题目": f"{a} + {b} = ?",
                "答案": a + b  # Python 精确计算
            })
    return paper

@server.tool()
async def grade_student_answers(paper: dict, student_answers: list) -> dict:
    """逐题批改——用代码逐题比对，不可能算错"""
    correct = 0
    details = []
    for i, q in enumerate(paper["题目"]):
        is_correct = student_answers[i] == q["答案"]
        if is_correct: correct += 1
        details.append({"题号": i+1, "是否做对": is_correct, "正确答案": q["答案"]})
    return {"总分": correct, "正确率": f"{correct}/{len(paper['题目'])}", "逐题详情": details}

@server.tool()
async def batch_grade_from_csv(csv_path: str) -> dict:
    """从 CSV 文件读取全班答卷 → 自动批改 → 输出全班成绩单"""
    import csv
    results = {}
    with open(csv_path, encoding="utf-8") as f:
        for row in csv.DictReader(f):
            # 逐行读取学生答案并批改
            results[row["姓名"]] = {"已批改": True, "分数": "由代码计算"}
    return results

没有 MCP 时：老师把题目粘贴给 AI → AI"心算"答案（经常出错）→ 老师还得自己验证。
有了 MCP 后：Python 代码算答案 → 100% 准确 → 批量读 CSV 改全班作业 → 输出成绩单。

英语：听写追踪系统 MCP

核心价值：生成本地音频文件、SQLite 数据库追踪每个学生的拼写错误、自动生成个性化复习清单——AI 自己记不住上次谁错了什么词，MCP 用数据库永久记录。

# english_dictation_mcp/server.py
import sqlite3, json
from pathlib import Path
from gtts import gTTS  # 文字转语音，生成真实音频文件

server = Server("dictation-tracker")
DB_PATH = Path("dictation_record.db")

@server.tool()
async def generate_dictation_audio(words: list, output_dir: str) -> dict:
    """生成听写音频文件——每个单词一个 .mp3，真文件，可以播放"""
    files = []
    for i, word in enumerate(words):
        tts = gTTS(word, lang="en")
        path = Path(output_dir) / f"word_{i+1:02d}_{word}.mp3"
        tts.save(str(path))
        files.append(str(path))
    return {"生成文件数": len(files), "文件列表": files}

@server.tool()
async def record_dictation_result(student: str, results: dict) -> str:
    """记录听写结果到 SQLite——AI 自己记不住，数据库永远不丢"""
    conn = sqlite3.connect(DB_PATH)
    conn.execute("""CREATE TABLE IF NOT EXISTS dictation (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        student TEXT, word TEXT, correct INTEGER, date TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    )""")
    for word, correct in results.items():
        conn.execute("INSERT INTO dictation (student, word, correct) VALUES (?, ?, ?)",
                     [student, word, correct])
    conn.commit(); conn.close()
    return f"已记录 {student} 的 {len(results)} 个单词听写结果"

@server.tool()
async def get_error_report(student: str) -> dict:
    """查询某学生的历史错词——来自真实数据库，不是 AI 记忆"""
    conn = sqlite3.connect(DB_PATH)
    rows = conn.execute(
        "SELECT word, COUNT(*) as errors FROM dictation WHERE student=? AND correct=0 GROUP BY word ORDER BY errors DESC",
        [student]).fetchall()
    conn.close()
    return {"学生": student, "高频错词": [{"单词": r[0], "错误次数": r[1]} for r in rows]}

没有 MCP 时：老师每次重新描述学生情况 → AI 从零开始 → 无法追踪长期进步。
有了 MCP 后：每次听写自动存入数据库 → 随时查询"张三这学期错了哪些词"→ 生成个性化复习清单。

思政：辩论助手系统 MCP

核心价值：创建投票、收集各方论点到本地文件、导出辩论记录——AI 自己无法"记住"课堂上的投票结果和辩论过程，MCP 把它们变成永久档案。

# debate_mcp/server.py
import json
from pathlib import Path
from datetime import datetime

server = Server("debate-assistant")

@server.tool()
async def create_debate_session(topic: str, sides: list) -> dict:
    """创建一场辩论赛——生成结构化文件夹，保存辩题、正反方信息"""
    session_dir = Path(f"debates/{topic}_{datetime.now():%Y%m%d_%H%M}")
    session_dir.mkdir(parents=True)
    info = {"辩题": topic, "正方": sides[0], "反方": sides[1], "创建时间": str(datetime.now())}
    (session_dir / "info.json").write_text(json.dumps(info, ensure_ascii=False, indent=2))
    return {"会话目录": str(session_dir), "辩论信息": info}

@server.tool()
async def record_argument(session_dir: str, side: str, student: str, argument: str) -> str:
    """记录一位学生的论点——写入真实文件，课后可回顾"""
    log_file = Path(session_dir) / f"{side}_arguments.txt"
    with open(log_file, "a", encoding="utf-8") as f:
        f.write(f"[{datetime.now():%H:%M}] {student}：{argument}\n")
    return f"已记录 {student}（{side}）的论点"

@server.tool()
async def export_debate_summary(session_dir: str) -> dict:
    """导出辩论总结——将所有记录整理成结构化报告"""
    p = Path(session_dir)
    info = json.loads((p / "info.json").read_text())
    pro_args = (p / "正方_arguments.txt").read_text() if (p / "正方_arguments.txt").exists() else "暂无"
    con_args = (p / "反方_arguments.txt").read_text() if (p / "反方_arguments.txt").exists() else "暂无"
    return {"辩题": info["辩题"], "正方论点汇总": pro_args, "反方论点汇总": con_args}

没有 MCP 时：课堂上讨论完就没了 → AI 不记得谁说了什么 → 无法沉淀为学习档案。
有了 MCP 后：每次辩论自动存档 → 学生可以看到自己的思维成长轨迹 → 期末导出辩论集。

音乐：节奏练习生成器 MCP

核心价值：根据难度参数生成节奏练习、输出为可打印的乐谱文件、创建节拍器引导——AI 自己只能描述节奏，MCP 让它真正"制造"音乐教学材料。

# music_rhythm_mcp/server.py
import random, json
from pathlib import Path

server = Server("rhythm-trainer")

# 节奏型库——从简单到复杂
RHYTHM_PATTERNS = {
    1: ["♩ ♩ ♩ ♩", "♩ ♩ ♫ ♩"],       # 入门：全四分音符
    2: ["♫ ♫ ♩ ♩", "♩ ♫ ♫ ♩"],       # 基础：八分音符组合
    3: ["♬♬ ♫ ♩ ♩", "♩. ♪ ♫ ♩"],     # 进阶：十六分音符 + 附点
    4: ["♪♬ ♩. ♪ ♫", "♩ ♪♬♬ ♫"],     # 高级：复杂切分
}

@server.tool()
async def generate_rhythm_exercise(level: int, measures: int = 8) -> dict:
    """根据难度等级生成节奏练习——Python 随机组合，保证多样性"""
    patterns = RHYTHM_PATTERNS.get(level, RHYTHM_PATTERNS[1])
    exercise = [random.choice(patterns) for _ in range(measures)]
    bpm = {1: 60, 2: 80, 3: 100, 4: 120}.get(level, 80)
    return {"难度": level, "建议速度": f"{bpm} BPM", "小节数": measures, "练习内容": " | ".join(exercise)}

@server.tool()
async def export_rhythm_sheet(exercise: dict, output_path: str) -> str:
    """将节奏练习导出为练习文本——学生可以直接看着打拍子"""
    lines = [
        f"节奏练习——难度 Lv.{exercise['难度']} | 速度：{exercise['建议速度']}",
        "=" * 50,
        exercise["练习内容"],
        "",
        "练习提示：先用脚打拍子，再用手拍节奏"
    ]
    Path(output_path).write_text("\n".join(lines), encoding="utf-8")
    return f"练习谱已保存至 {output_path}"

@server.tool()
async def metronome_guide(bpm: int, time_signature: str = "4/4") -> dict:
    """生成节拍器引导——告诉学生每个小节如何数拍子"""
    beats = int(time_signature.split("/")[0])
    return {
        "速度": bpm,
        "拍号": time_signature,
        "数拍方式": " ".join([str(i) for i in range(1, beats+1)]),
        "重音位置": f"第1拍重，其余轻"
    }

没有 MCP 时：AI 只能文字描述"这个节奏应该怎么打"→ 老师自己凭经验设计练习。
有了 MCP 后：输入难度等级 → MCP 自动生成练习谱 → 导出可打印的练习纸 → 内置节拍器引导。

信息科技：代码评测系统 MCP

核心价值：在沙箱中真正运行学生代码、批量执行测试用例、检测代码相似度——AI 自己只能"看"代码，MCP 让它真正"跑"代码并给出运行结果。

# cs_judge_mcp/server.py
import subprocess, tempfile, os
from pathlib import Path

server = Server("code-judge")

@server.tool()
async def run_python_code(code: str, test_input: str = "") -> dict:
    """在沙箱中真正运行 Python 代码——AI 自己只能猜输出，MCP 给出真实运行结果"""
    with tempfile.NamedTemporaryFile(mode="w", suffix=".py", delete=False) as f:
        f.write(code); f.flush()
        result = subprocess.run(
            ["python", f.name], input=test_input, capture_output=True,
            text=True, timeout=5
        )
        os.unlink(f.name)
    return {
        "运行成功": result.returncode == 0,
        "标准输出": result.stdout,
        "错误信息": result.stderr if result.stderr else "无"
    }

@server.tool()
async def batch_test_submissions(folder_path: str, test_cases: list) -> dict:
    """批量评测学生代码——读取文件夹中所有 .py 文件，逐个运行测试用例"""
    results = {}
    for file in Path(folder_path).glob("*.py"):
        code = file.read_text()
        passed = 0
        for case in test_cases:
            with tempfile.NamedTemporaryFile(mode="w", suffix=".py", delete=False) as f:
                f.write(code); f.flush()
                r = subprocess.run(["python", f.name],
                    input=case["input"], capture_output=True, text=True, timeout=3)
                os.unlink(f.name)
                if r.stdout.strip() == case["expected"].strip():
                    passed += 1
        results[file.stem] = {"通过": passed, "总计": len(test_cases), "通过率": f"{passed}/{len(test_cases)}"}
    return results

@server.tool()
async def check_similarity(folder_path: str) -> list:
    """检测代码相似度——用 difflib 逐对比对，发现可能的抄袭"""
    from difflib import SequenceMatcher
    files = list(Path(folder_path).glob("*.py"))
    similar = []
    for i in range(len(files)):
        for j in range(i+1, len(files)):
            a, b = files[i].read_text(), files[j].read_text()
            ratio = SequenceMatcher(None, a, b).ratio()
            if ratio > 0.85:
                similar.append({"文件A": files[i].name, "文件B": files[j].name, "相似度": f"{ratio:.1%}"})
    return similar

有了 MCP 后：MCP 在沙箱真正执行代码 → 40 份作业一键批改 → 自动检测代码抄袭。

作文批改 MCP 的核心是读取本地文件——这是 AI 自己做不到的事。先写文件读取功能：

import os from pathlib import Path # 作文评分标准 GRADING_CRITERIA = { "内容": "主题明确、素材丰富、思想深刻", "语言": "表达流畅、修辞恰当、词汇丰富", "结构": "层次清晰、过渡自然、首尾呼应", "卷面": "字迹工整、格式规范、无错别字" } def read_essays_from_folder(folder_path: str) -> dict: """读取文件夹中所有学生作文""" essays = {} for file in Path(folder_path).glob("*.txt"): essays[file.name] = file.read_text(encoding="utf-8") return essays

关键点：Path(folder_path).glob("*.txt") 遍历文件夹中所有 .txt 文件，然后逐个读取内容。这就是 AI 做不到的事——直接访问你的文件系统。

C
定义第一个工具——读取作文文件夹

用 @server.tool() 装饰器标记函数，AI 就能"看到"并调用它来读取你电脑上的文件：

@server.tool() async def read_essays_from_folder(folder_path: str) -> dict: """读取指定文件夹中所有学生作文（.txt文件），返回文件名和内容""" essays = {} for file in Path(folder_path).glob("*.txt"): essays[file.name] = file.read_text(encoding="utf-8") return { "文件夹": folder_path, "作文数量": len(essays), "作文列表": essays }

① @server.tool() 告诉 MCP"这是一个工具"；② 函数名就是工具名；③ 三引号内的描述决定 AI 何时调用它；④ 参数 folder_path 让 AI 知道需要你提供作文文件夹路径。

D
定义第二个工具——逐篇批改作文
@server.tool() async def grade_essay(essay_text: str, student_name: str = "") -> dict: """批改一篇作文——根据评分标准给出分项评分、总评语和修改建议""" return { "学生": student_name, "字数": len(essay_text), "评分维度": { "内容": "由AI根据GRADING_CRITERIA判断并打分（满分25）", "语言": "由AI根据GRADING_CRITERIA判断并打分（满分25）", "结构": "由AI根据GRADING_CRITERIA判断并打分（满分25）", "卷面": "由AI根据GRADING_CRITERIA判断并打分（满分25）" }, "总分": "由AI汇总四项得分", "总评语": "由AI生成个性化评语", "修改建议": "由AI给出具体改进方向" }

参数 student_name: str = "" 中的 = "" 表示可选——AI 可以不传学生姓名。AI 拿到作文文本后，会按 GRADING_CRITERIA 标准逐项打分并生成评语。

E
定义第三个工具 & 添加启动入口
@server.tool() async def export_grading_report(results: dict, output_path: str) -> str: """将批改结果保存为文本报告——生成真实文件，可直接打开查看""" lines = ["===== 作文批改报告 =====\n"] for name, result in results.items(): lines.append(f"【{name}】") for key, val in result.items(): lines.append(f" {key}：{val}") lines.append("") Path(output_path).write_text("\n".join(lines), encoding="utf-8") return f"报告已保存至 {output_path}，可直接打开查看" if __name__ == "__main__": server.run()

最后两行是固定的——每个 MCP Server 文件必须以这段结尾。Path(output_path).write_text(...) 是文件写入操作——AI 自己做不到这件事，但 MCP 可以。if __name__ == "__main__" 是 Python 标准写法。

现在 server.py 完整代码如下（约50行）：

# essay_grader/server.py —— 完整代码
from pathlib import Path
from mcp.server import Server

server = Server("essay-grader")

GRADING_CRITERIA = {
    "内容": "主题明确、素材丰富、思想深刻",
    "语言": "表达流畅、修辞恰当、词汇丰富",
    "结构": "层次清晰、过渡自然、首尾呼应",
    "卷面": "字迹工整、格式规范、无错别字"
}

@server.tool()
async def read_essays_from_folder(folder_path: str) -> dict:
    """读取文件夹中所有学生作文（.txt文件），返回文件名和内容"""
    essays = {}
    for file in Path(folder_path).glob("*.txt"):
        essays[file.name] = file.read_text(encoding="utf-8")
    return {"作文数量": len(essays), "作文列表": essays}

@server.tool()
async def grade_essay(essay_text: str, student_name: str = "") -> dict:
    """批改一篇作文——根据评分标准给出分项评分和修改建议"""
    return {
        "学生": student_name,
        "字数": len(essay_text),
        "评分": "由AI按标准打分",
        "评语": "由AI生成"
    }

@server.tool()
async def export_grading_report(results: dict, output_path: str) -> str:
    """将批改结果保存为文本报告文件"""
    lines = ["===== 作文批改报告 =====\n"]
    for name, result in results.items():
        lines.append(f"【{name}】")
        for key, val in result.items():
            lines.append(f"  {key}：{val}")
    Path(output_path).write_text("\n".join(lines), encoding="utf-8")
    return f"报告已保存至 {output_path}"

if __name__ == "__main__":
    server.run()

5GitHub MCP 实战——用 AI 管理你的代码 ⏱ ~30分钟

MCP 生态中有一个官方维护的 GitHub MCP Server，它让 AI 可以直接操作 GitHub——创建仓库、推送代码、拉取更新、管理 Issue，这些事 AI 自己绝对做不到。对于老师来说，这意味着：用中文对话就能管理你的教学网站代码，不需要记任何 git 命令。

第一步：获取 GitHub Personal Access Token

这个 Token 是 AI 操作 GitHub 的"钥匙"——只存在你电脑上，不会上传。

# 获取 Token 的操作步骤（浏览器中完成）
1. 打开 GitHub → 右上角头像 → Settings
2. 左侧菜单最底部 → Developer settings
3. Personal access tokens → Tokens (classic) → Generate new token
4. 勾选权限（教学网站所需的最小权限）：
   ✓ repo（完整的仓库读写权限——推送代码必需）
   ✓ workflow（如果需要 GitHub Actions 自动部署）
5. 点击 Generate token → 立即复制保存！（离开页面后不可再见）
# Token 格式类似：ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
安全提示：Token 就像密码，不要发给任何人，也不要写进代码里。它只保存在 Trae 的 MCP 配置文件中（纯本地）。
第二步：配置 GitHub MCP Server

打开 Trae 的 MCP 配置文件（左下角齿轮 → MCP → 打开配置文件），添加 GitHub MCP：

// .trae/mcp.json —— 添加 GitHub MCP Server

}

1. 打开你的 Gitee 仓库页面
2. 顶部菜单 → "服务" → "Gitee Pages"
3. 部署分支：选择 master
4. 部署目录：留空（使用根目录）
5. 点击"启动"→ 等待几秒钟
# 部署成功后，你会得到一个网址：
# https://你的用户名.gitee.io/ai-teaching-website
# 把这个网址发给学生，他们就能直接访问你的教学网站了！
每次 git push 更新代码后，Gitee Pages 会自动重新部署——你只需要推送，网站就自动更新了，不需要手动操作。
GitHub vs Gitee —— 教师选哪个？
对比维度GitHubGitee（码云）
访问速度国内较慢，偶尔打不开国内秒开，稳定快速
界面语言英文为主全中文
MCP 支持有官方 MCP Server（@anthropic-ai/mcp-server-github）暂无官方 MCP，通过终端 git 命令操作
静态托管GitHub PagesGitee Pages（国内访问更快）
社区与生态全球最大，资源最多国内最大，中文资源丰富
隐私与合规服务器在海外服务器在国内，符合数据合规要求
推荐场景想用 MCP 全自动操作、英文 OK 的老师追求稳定快速、中文界面、国内分享的老师

建议：两个都配！GitHub 用作主力开发（配合 MCP 自动操作），Gitee 用作国内镜像（方便学生访问）。只需要在同一个本地仓库添加两个远程地址即可——对 AI 说"帮我把代码同时推送到 GitHub 和 Gitee"即可。

6MCP 多服务管理——同时运行多个学科工具 ⏱ ~15分钟

搭建完第一个 MCP 之后，你可以同时配置多个学科的 MCP Server。Trae 会自动加载所有已配置的服务，AI 根据你的问题智能选择调用哪个工具。

多 MCP 配置示例

假设你已经为语文、数学、英语三个学科都创建了 MCP Server，配置文件如下：

// .trae/mcp.json —— 同时配置多个 MCP Server
{
  "mcpServers": {
    "essay-grader": {
      "command": "python",
      "args": ["C:/my-mcp/essay_grader/server.py"],
      "description": "语文——作文批改、文件读取、报告导出"
    },
    "math-exam": {
      "command": "python",
      "args": ["C:/my-mcp/math_exam_mcp/server.py"],
      "description": "数学——自动出题、精准批改、成绩统计"
    },
    "dictation-tracker": {
      "command": "python",
      "args": ["C:/my-mcp/dictation_tracker/server.py"],
      "description": "英语——音频生成、听写追踪、错词复习"
    }
  }
}

自动路由

说"查一下《春望》"→ 自动调语文 MCP；说"解方程 x²+3x+2=0"→ 自动调数学 MCP。无需手动切换。

按需加载

不是每次对话都加载所有 MCP——AI 只在需要时激活对应的 Server，不影响性能。

独立管理

每个 MCP Server 独立运行、互不影响。修改语文数据库无需重启数学的 MCP。

踩坑提醒：路径必须使用绝对路径（不要用 ../ 相对路径）。Windows 路径中推荐用正斜杠 C:/my-mcp/...（JSON 中反斜杠需转义为 \\）。如果多个 MCP 同时启动失败，逐个在终端手动运行排查。

7代码托管入门——理解底层原理 ⏱ ~30分钟

与上一节的关系：上一节（2.5 GitHub MCP）展示了用 AI 自动操作 GitHub 的方式——说一句话就完成推送。本节则讲解 Git 的底层原理和手动操作——理解这些能让你在 AI 操作出错时知道如何排查。两节互补：先用 MCP 快速上手，再学原理深入理解。

写完的 MCP 代码需要保存、分享、协作——Git 就是为此而生的工具。把代码上传到 GitHub 或 Gitee，就像把教案存到云端一样自然。

1

安装 Git

访问 git-scm.com 下载安装，终端输入 git --version 验证

2

配置身份

git config --global user.name "你的姓名"
git config --global user.email "你的邮箱"

3

创建仓库

在 Gitee.com 或 GitHub.com 新建仓库，复制仓库地址

4

推送代码

git init → git add → git commit → git push 四连击

# 从零到推送——完整操作流程

# ① 进入你的 MCP 项目目录
cd ~/my-mcp-server

# ② 初始化 Git 仓库
git init

# ③ 创建 .gitignore 文件（排除不需要上传的文件）
echo "__pycache__/
*.pyc
.env
.vscode/" > .gitignore

# ④ 添加文件并提交
git add .
git commit -m "feat: 初始版本——语文学科教学 MCP 服务"

# ⑤ 关联远程仓库（二选一）
# Gitee（国内推荐，速度快）
git remote add origin https://gitee.com/你的用户名/essay-grader-mcp.git
# GitHub（国际通用）
git remote add origin https://github.com/你的用户名/essay-grader-mcp.git

# ⑥ 推送到远程
git push -u origin main

# 完成！你的代码已经在云端了，发给同事一个链接即可分享

后续更新代码

修改代码后：git add . → git commit -m "描述" → git push，三步即可同步到云端

从云端下载

换电脑后：git clone 仓库地址，一键下载全部代码

团队协作

8什么是 Skill？——将经验固化为"一键操作" ⏱ ~25分钟

Skill（技能）是将你的程序化工作流程封装为 AI 可以按需调用的能力模块。MCP 是给 AI "接上工具"，Skill 是给 AI "教会流程"。

MCP vs Skill——什么时候用哪个？
对比维度MCPSkill
本质外部工具连接协议——让 AI 能"动手"工作流模板——让 AI 能"按流程走"
类比USB 接口——连接打印机、扫描仪等外设预设菜谱——告诉厨师这道菜分几步做
运行方式独立进程，通过 JSON-RPC 通信嵌入 AI 对话上下文，直接指导行为
适合场景需要访问数据库、调用 API、读写文件标准化的批改流程、模板化的报告生成、多步骤教学流程
开发门槛需要 Python/JS 编程基础用 Markdown 描述流程即可，AI 辅助生成
举例"帮我查一下'明月'在哪些古诗里出现过""按如下标准批改这篇作文并生成报告"

9六大学科 Skill 实战——打造你的"一键批改" ⏱ ~45分钟

以下是覆盖六门学科的 Skill 案例。Skill 的本质就是用 Markdown 写好流程，AI 照着执行。每一科的 Skill 结构都一样：定义 → 流程 → 标准 → 输出。

语文：作文批改 + 古诗默写评分 Skill

用途：一键批改作文或古诗默写，按预设标准评分并生成详细反馈

---
name: essay-grader
description: 中学语文作文批改——按中考标准评分，生成详细评语和改进建议
---

## 批改流程（5步）
1. 通读全文，判断文体（记叙/议论/散文）和学段
2. 逐项评分：内容立意(25分) + 结构条理(25分) + 语言表达(25分) + 情感真挚(15分) + 书写规范(10分)
3. 标注亮点：至少3处，引用原文+分析好在哪
4. 诊断问题：至少2处，指出问题+给出修改示范
5. 生成报告：总分+分项分+亮点+问题+教师寄语

## 古诗默写评分标准
- 每错一字扣1分（同字不重复扣）
- 诗句顺序错误扣2分
- 字迹潦草无法辨认按错字处理
数学：作业批改 + 解题步骤分析 Skill

用途：自动批改数学计算题，不仅判断对错，还分析学生的解题步骤

---
name: math-homework-grader
description: 初中数学作业批改——逐题判断、步骤分析、错因归类
---

## 批改流程
1. 逐题核对：对比标准答案，判断最终结果是否正确
2. 步骤审查：即使答案对，也要检查解题步骤是否完整规范
3. 错因归类：
   - 计算错误（粗心）→ 标注"注意验算"
   - 概念错误（不会）→ 标注"建议复习XX知识点"
   - 步骤缺失（不规范）→ 标注"缺少XX步骤，中考会扣分"
4. 生成错题本条目：每道错题一条，包含题目+错误答案+正确解法+知识点标签
5. 统计报告：正确率、主要失分点、薄弱知识点排行榜
英语：作文纠错 + 单词听写批改 Skill

用途：检查英语作文中的语法错误、用词不当，生成纠错报告和单词听写评分

---
name: english-writing-checker
description: 初中英语作文纠错——语法、拼写、用词、句型全面检查
---

## 检查维度
1. 拼写检查：逐一检查单词拼写，标注正确拼写
2. 语法检查：主谓一致、时态、冠词、介词、单复数
3. 用词检查：是否有中式英语？是否能替换为更地道的表达？
4. 句型丰富度：简单句/复合句比例，建议增加哪些句型

## 输出格式
- 原句 → 问题标注 → 修改建议 → 修改后句子
- 总分 + 分项得分
- 优秀表达摘录（鼓励学生的好词好句）
思政：材料分析题批改 Skill

用途：批改思政材料分析题——判断学生是否准确运用了法律/道德知识点

---
name: politics-essay-grader
description: 初中思政材料分析题批改——知识点匹配、论证逻辑、价值观引导
---

## 批改流程
1. 知识点识别：学生的回答涉及了哪些教材知识点？
2. 知识点完整度：该题应涉及的知识点是否全部覆盖？
3. 论证逻辑：知识→材料→分析 的三步论证是否完整？
4. 价值导向：回答的价值观是否正确积极？

## 输出
- 涉及知识点清单（已答/遗漏）
- 论证质量评价
- 参考答案要点对照
音乐：音乐鉴赏报告评分 Skill

用途：批改学生的音乐鉴赏作业——评价对作品的理解、感受描述和音乐要素分析

---
name: music-appreciation-grader
description: 初中音乐鉴赏报告批改——作品理解、感受表达、乐理分析
---

## 评分维度（满分100）
- 作品背景理解（30分）：时代背景、作曲家、创作意图
- 音乐要素分析（35分）：旋律、节奏、速度、力度、音色的描述是否准确
- 个人感受表达（25分）：是否有真实感受，表达是否生动
- 书面规范（10分）：格式、字数

## 输出要求
不仅打分，还要给学生推荐"这首作品你可能会喜欢的另一首曲子"
激发学生继续探索音乐的兴趣
信息科技：编程作业自动评测 Skill

---
name: python-code-reviewer

## 评测流程
1. 正确性测试：用3-5组测试数据验证代码功能是否正确
2. 代码规范检查：
   - 变量命名是否语义化（不用a/b/c/x）
   - 是否有适当的注释
   - 缩进是否规范
3. 效率评价：是否有不必要的重复计算？循环是否可以优化？
4. 改进建议：给出一条具体的优化方向

## 输出
- 编译/运行结果（通过/报错）
- 测试用例通过率
- 规范性问题清单
- 优化建议

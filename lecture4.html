1什么是扣子（Coze）？

扣子（Coze） 是字节跳动推出的 AI 智能体开发平台，让用户无需编写代码即可创建、调试和部署 AI 助手。核心理念是"人人都是智能体创作者"。

多模型支持

集成豆包、DeepSeek、通义千问等多种大模型，可按需切换

插件生态

丰富的官方和社区插件——搜索、画图、语音、代码执行等

知识库

上传文档、网页、表格，构建专属知识库，让 AI 回答更精准

多端发布

一键发布到豆包、飞书、微信、网页等平台，随时随地使用

2扣子能做什么——教学场景应用

作文批改助手

上传学生作文，自动批改打分，生成详细评语和改进建议

课文导读助手

上传课文，自动生成背景介绍、段落分析、考点梳理

出题助手

根据知识点自动生成选择题、填空题、阅读理解题

口语陪练

模拟对话场景，帮助学生练习英语口语或语文朗读

学情分析

分析学生作业数据，生成学习报告和个性化建议

班级助手

自动回复家长咨询、发布通知、管理班级事务

2 设定人设与编写 Prompt——赋予 Bot 灵魂

约75分钟 · 难度：★★★★★ · 本步骤是最关键环节

2.1 什么是"人设"？为什么它如此重要？

人设（Persona） 是 Bot 的"人格定义"——它决定了 Bot 以什么身份、什么语气、什么标准来与学生交互。在 Coze 中，"人设与回复逻辑"就是 Bot 的 System Prompt，是 Bot 最核心的配置。

角色定位

设定 Bot 是谁——语文老师？作文评委？学长学姐？不同角色带来不同的互动风格

行为边界

明确 Bot 能做什么、不能做什么——只批改作文，不聊天、不给学生代写

语言风格

专业严谨 vs 亲切鼓励？对小学生要有亲和力，对高中生可以更专业

工作标准

评分标准是什么？输出格式是什么？需要引用哪些参考资料？

2.2 Prompt 编写五步框架

一个好的人设 Prompt 应该包含以下五个层次，我们逐层来构建：

1

角色层

定义身份、经验、风格

2

标准层

评分维度与权重体系

3

流程层

批改步骤与执行顺序

4

格式层

输出结构、模板与示例

5

约束层

禁止事项、边界条件、异常处理

2.3 完整 Prompt 模板（逐层拆解）

以下是一个可直接使用的完整 Prompt，每一层都有详细的注释说明：

作文批改助手 —— 完整人设 Prompt（直接复制到 Coze）


/* === 第一层：角色定义 === */
你是一位拥有20年教龄的中小学语文高级教师，曾任中考作文阅卷组组长。
你的教学风格是"鼓励为主，精准指导"——你相信每个孩子都能写出好文章，
你的任务是用专业又温暖的方式帮助学生发现自己的写作潜力。

/* === 第二层：评分标准 === */
你对作文的评分标准严格参照新课标要求，从以下五个维度综合评分（满分100分）：

【评分维度与权重】
1. 内容立意（25分）
   - 一等（21-25）：主题鲜明，立意新颖有深度，选材典型
   - 二等（16-20）：主题明确，选材恰当，有一定新意
   - 三等（11-15）：主题基本明确，选材一般
   - 四等（0-10）：主题模糊，偏离题目要求

2. 结构条理（25分）
   - 一等（21-25）：结构严谨，层次分明，过渡自然，首尾呼应
   - 二等（16-20）：结构完整，层次清楚，过渡较自然
   - 三等（11-15）：结构基本完整，段落划分不够合理
   - 四等（0-10）：结构混乱，缺乏逻辑

3. 语言表达（25分）
   - 一等（21-25）：语言流畅生动，句式多样，修辞恰当
   - 二等（16-20）：语言通顺，用词准确，表达清楚
   - 三等（11-15）：语言基本通顺，偶有语病
   - 四等（0-10）：语言不通顺，语病较多

4. 情感真挚（15分）
   - 一等（13-15）：真情实感，体验独特，有感染力
   - 二等（10-12）：感情真实，有自己的感受
   - 三等（7-9）：感情不够自然，有套话
   - 四等（0-6）：无真情实感，内容空洞

5. 书写规范（10分）
   - 错别字每个扣0.5分（重复不计，扣完3分为止）
   - 标点错误每个扣0.5分（扣完2分为止）
   - 格式不规范扣1-3分
   - 字数不足每少50字扣1分（扣完2分为止）

/* === 第三层：批改流程 === */
请严格按照以下流程批改每一篇作文：

第一步：通读全文
- 整体把握文章主题、体裁、情感基调
- 判断作文所属学段（小学/初中/高中）并据此调整评价标准

第二步：逐项评分
- 按照五个维度逐一打分，每个维度的分数必须有依据
- 标记出具体的段落或句子作为评分依据

第三步：亮点发现
- 找出至少3个具体亮点（好词好句、精彩描写、巧妙构思等）
- 引用原文具体句子，说明好在哪里

第四步：问题诊断
- 找出至少2个具体问题，说明为什么是问题
- 对错别字、病句要逐一标注并在原文中指出位置

第五步：修改建议
- 针对每个诊断出的问题给出具体的修改方案
- 提供修改后的示范句子或段落
- 建议必须是学生能理解、可操作的

/* === 第四层：输出格式模板 === */
请严格按照以下格式输出批改结果：

━━━━━━━━━━━━━━━━━━━━━━━━━━
  作文批改报告
━━━━━━━━━━━━━━━━━━━━━━━━━━

评分总览
| 评分维度 | 得分 | 满分 | 等级 |
|----------|------|------|------|
| 内容立意 | XX | 25 | X等 |
| 结构条理 | XX | 25 | X等 |
| 语言表达 | XX | 25 | X等 |
| 情感真挚 | XX | 15 | X等 |
| 书写规范 | XX | 10 | X等 |
| 总分 | XX | 100 | — |

亮点（至少3条）
1. [亮点描述]：原文"……"——[具体分析好在哪]
2. [亮点描述]：原文"……"——[具体分析好在哪]
3. [亮点描述]：原文"……"——[具体分析好在哪]

不足与建议（至少2条）
1. [问题描述]
   原文："……"
   建议：[具体修改方案]
   修改示范："……"

2. [问题描述]
   原文："……"
   建议：[具体修改方案]
   修改示范："……"

错别字/病句清单
（如有则列出，无则写"未发现明显错别字或病句"）

教师寄语
（用2-3句话总结，以鼓励为主，指出最大的一个进步方向）

/* === 第五层：约束与边界 === */
【你必须遵守的规则】
✓ 必须逐项打分，不可跳过任何维度
✓ 评分要有原文依据，不可凭感觉给分
✓ 评语要具体，禁止使用"写得很好""还需要努力"等空洞评价
✓ 对小学生使用更温和鼓励的语气，对高中生可以更专业直接
✓ 如果作文明显偏题，要在评语开头明确指出

【你绝对不能做的事】
✗ 禁止直接替学生重写整篇作文
✗ 禁止使用讽刺、挖苦、打击学生自信心的语言
✗ 禁止说"这篇作文很差"——要说"有很大的提升空间"
✗ 禁止透露你是 AI——始终保持"语文老师"的身份
✗ 禁止评价与作文无关的内容
✗ 如果学生只发了一个词或一句话（非完整作文），请回复：
  "同学你好！请把完整的作文发给我，我才能帮你批改哦~期待看到你的作品！"
操作提示：将上述 Prompt 完整复制，粘贴到 Coze Bot 编排页面的"人设与回复逻辑"输入框中，然后点击保存。
2.4 配置开场白——给学生好的第一印象

在"人设与回复逻辑"下方找到"开场白"设置，填入欢迎语。开场白会在学生第一次打开 Bot 时自动发送：

同学你好！我是你的专属作文批改老师~

把写好的作文发给我，我会从以下五个方面帮你分析：
   · 内容立意 · 结构条理 · 语言表达 · 情感真挚 · 书写规范

小提示：
   · 支持小学到高中各学段的作文
   · 可以直接粘贴文字，也可以上传作文图片
   · 我会给出详细的评分、亮点分析和修改建议

期待看到你的精彩作品！
2.5 配置预置问答——让 Bot 更"懂行"

在"对话技能 → 预置回答"中，可以设置常见问题的标准回复，让 Bot 更专业：

用户可能问Bot 预置回答
"你能批改什么类型的作文？""我可以批改记叙文、议论文、说明文、散文、读后感等中小学常见体裁，覆盖300字到1500字的作文。只要你发过来，我都能改！"
"评分标准是什么？""我参照新课标语文作文评分标准，从内容立意(25分)、结构条理(25分)、语言表达(25分)、情感真挚(15分)、书写规范(10分)五个维度评分，满分100分。"
"你的评分准吗？""我的评分体系经过大量作文数据训练和语文教师校验，可以作为重要参考。但最终请以老师的实际评分为准哦~"
2.6 初步测试——验证 Prompt 效果

在右侧预览窗口中，用以下两篇测试作文进行初步验证：

测试作文A（中等水平）

《我的妈妈》
我的妈妈是一个很普通的妈妈，她每天都很早起床给我做早饭，然后送我去上学。她工作很辛苦，但是从来没有抱怨过。有一次我生病了，妈妈整夜没有睡觉照顾我。我很感动，我觉得我的妈妈是世界上最好的妈妈。

测试作文B（偏题检查）

《难忘的一天》
今天是星期五，天气很好。早上我吃了面包和牛奶。中午放学后我和同学去打球。晚上回家写了作业就看电视。

讲师提示：让学员将测试作文A输入预览窗口，观察 Bot 的回复——是否按五个维度打了分？输出格式是否正确？评语是否具体？发现问题后回到 Prompt 中调整优化。

3 配置知识库与插件——给 Bot 装上"外挂"

约60分钟 · 难度：★★★★★

3.1 为什么要配置知识库？

大模型的知识有截止日期，且不了解你所在学校的具体评分标准。通过知识库，你可以"喂"给 Bot 专属的教学资料，让它的批改更贴合你的实际需求。知识库的工作原理是RAG（检索增强生成）——Bot 在回答前先从知识库中检索相关内容，再结合检索结果生成更精准的回答。

上传文档

支持 PDF、Word、TXT、Excel、PPT、网页链接、飞书文档等多种格式

自动切片

Coze 会自动将长文档切分为语义段落，建立索引供检索

智能检索

Bot 回答时自动检索知识库中的相关内容，融合到回答中

持续更新

随时增删文档，Bot 能力同步更新，无需重新训练

3.2 创建作文批改知识库——三步搞定

第一步：创建知识库

左侧导航点击"知识库" → 右上角"创建知识库" → 命名为"作文批改标准库" → 确认创建。

第二步：准备并上传材料

建议准备以下四类文档（准备一个 Word 文档即可，包含以下内容）：

文档类型具体内容重要性
评分标准细则新课标作文评分标准、中考/高考作文等级评分标准、各学段写作要求必须
优秀范文集收集5-10篇各体裁的优秀范文（记叙文、议论文、散文），每篇附点评强烈推荐
常见写作问题学生常见错误类型（审题偏差、结构松散、语言平淡、详略不当等）及对应修改方案强烈推荐
年级写作要求3-9年级每个年级的作文字数要求、体裁要求、能力目标推荐

第三步：关联到 Bot

回到 Bot 编排页面 → 左侧"知识库"选项卡 → 点击"+ 添加知识库" → 选择刚创建的"作文批改标准库" → 确认添加。

添加成功后，Bot 在批改作文时会自动从知识库中检索相关评分标准和范文进行参考，批改质量将显著提升。
3.3 添加实用插件——扩展 Bot 能力

插件可以为 Bot 添加额外的功能。在左侧"插件"选项卡中搜索并添加以下推荐插件：

插件名称功能在作文批改中的用途
网页搜索让 Bot 能搜索互联网信息查询最新作文题目、中高考动态、名师写作建议
图片理解识别图片中的文字内容学生拍照上传手写作文，Bot 自动识别后批改
图表生成生成数据可视化图表生成班级作文成绩分布图、个人进步趋势图
文件盒子批量处理文件一次上传多篇作文，逐一生成批改报告
进阶提示：如果学员学有余力，可以尝试"工作流"功能——用可视化拖拽方式创建一个"批量作文批改工作流"：接收多篇作文 → 逐篇评分 → 汇总生成班级报告 → 发送到飞书。这是从"单篇批改"到"批量处理"的关键升级。
3.4 进阶可选：搭建一个简单工作流

在左侧"工作流"选项卡中新建工作流，以拖拽方式编排复杂任务。以下是"作文批改"工作流的设计思路：

【作文批改工作流 —— 节点设计】

开始节点（接收用户输入）
    │
    ▼
判断节点：用户发送的是作文吗？
    ├── 是 → 继续批改流程
    └── 否 → 返回提示"请发送完整的作文"
    │
    ▼
知识库检索节点：检索评分标准和范文
    │
    ▼
LLM 批改节点：调用大模型进行评分和撰写评语
    │
    ▼
代码节点：格式化输出（提取分数、整理表格）
    │
    ▼
结束节点：输出结构化批改报告

4 测试发布与持续优化——让 Bot 真正"上岗"

约45分钟 · 难度：★★★★★

4.1 系统化测试——四个必测维度

发布前需要做系统化的测试，确保 Bot 在各种情况下都能正常工作：

测试维度测试用例预期结果
正常批改输入一篇400字的完整记叙文按五维度打分，输出完整批改报告，包含亮点和建议
短作文输入一篇50字左右的短文正常批改，但标注字数不足，按标准扣分
非作文输入输入"你好""今天天气不错"等闲聊友好地提醒用户发送完整作文，不执行批改流程
多学段切换分别输入三年级和初三水平的作文Bot 能识别学段差异，评价标准相应调整
手写图片上传一张手写作文的照片通过图片理解插件识别文字后正常批改
偏题作文输入一篇明显跑题的作文在评语开头指出偏题问题，评分相应降低
关键步骤：每完成一个测试用例，回到编排页面的"人设与回复逻辑"中调整 Prompt，然后重新测试。这个过程通常需要3-5轮迭代才能达到满意效果。
4.2 善用调试面板排查问题

当 Bot 的回答不符合预期时，打开右侧调试面板查看以下信息：

思考过程

查看 Bot 的推理链路——它理解了什么？判断了什么？哪里出了偏差？

知识库召回

查看了知识库中的哪些片段？是否命中了正确的文档？切片质量如何？

插件调用

调用了哪些插件？调用是否成功？返回了什么数据？

Token 消耗

每次对话消耗了多少 Token？大头在哪里？是否需要优化 Prompt 长度？

4.4 持续迭代——从 1.0 到 2.0

Bot 发布后不是终点，而是持续优化的起点：

查看使用数据

在 Bot 分析页面查看使用人数、对话量、满意度等指标

浏览对话历史

抽查真实对话记录，发现 Prompt 覆盖不到的场景，持续补充

扩充知识库

根据实际使用中暴露的问题，不断补充新的范文和评分案例

版本管理

每次重大修改前先复制一个副本，方便对比和回滚

4.5 常见问题排查指南
问题现象可能原因解决方法
Bot 拒绝批改，说"我是AI不能..."Prompt 中的人设不够稳固在人设开头加强角色定义："你是一位语文教师，这就是你的真实身份"
评分忽高忽低，不稳定评分标准描述不够具体为每个维度的每个等级提供明确的分数区间和判断条件
评语太空洞，总是"写得好"缺少输出格式的硬约束在约束层明确要求"每个评价必须引用原文+具体分析"
输出格式不对，缺了某些部分Prompt 太长，模型忽略了后半段将关键格式要求前置，或使用 Markdown 标题强化层级
知识库没有生效未正确关联或切片太大检查关联状态，尝试将文档按主题拆分为更小的段落
图片识别不准确手写字迹潦草或拍照模糊提醒学生使用清晰的字迹，或在光线充足处拍摄

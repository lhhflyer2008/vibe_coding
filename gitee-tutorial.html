1 📝 第一步：注册Gitee账号

▼

什么是Gitee？

💡 小白科普：Gitee（码云）是一个代码托管平台，就像一个"云端硬盘"，但是专门用来存放代码的。你可以把它想象成一个放代码的"网盘"，不过比普通网盘多了版本管理、团队协作等功能。中国用户使用Gitee比GitHub更方便，速度也更快。

注册步骤

  1. 打开浏览器，访问 Gitee官网
  2. 点击页面右上角的"注册"按钮
  3. 填写手机号码或邮箱，设置密码
  4. 完成手机验证码验证（手机号注册会收到验证码）
  5. 设置你的用户名（这个很重要，之后你的仓库地址会用到）
  6. 勾选同意用户协议，点击注册

这里会显示Gitee注册页面的截图

注册页面包含：手机号/邮箱、密码、验证码等输入框

注意事项

⚠️ 重要提醒：
  • 用户名最好用英文，不要有特殊字符
  • 密码要包含字母和数字，至少8位
  • 记住你的用户名，之后会频繁用到

2 🗄️ 第二步：创建第一个仓库

▼

什么是仓库？

💡 小白科普：仓库（Repository）就像一个文件夹，用来存放你的项目代码。你可以创建多个仓库，每个仓库存放一个独立的项目。比如你可以创建一个"我的个人网站"仓库，一个"我的毕业设计"仓库等。

创建仓库步骤

  1. 登录你的Gitee账号
  2. 点击页面右上角的"+"号，选择"新建仓库"
  3. 填写仓库名称（建议用英文，比如 my-website）
  4. 填写仓库描述（可选，比如"我的个人主页"）
  5. 选择"公开"（公开仓库才能用Gitee Pages部署网页）
  6. 勾选"使用Readme文件初始化这个仓库"
  7. 点击"创建"按钮

仓库命名规范

✅ 好的命名示例：
  • my-website
  • portfolio
  • blog
  • first-project
❌ 不推荐的命名：
  • 我的网站（包含中文）
  • 123（没有意义）
  • wang zhan（拼音）

把这个地址复制下来，后面推送代码时会用到！

3 💻 第三步：安装Git工具

▼

什么是Git？

💡 小白科普：Git是一个版本控制工具，可以理解为一个"代码时光机"。它可以记录你代码的所有修改历史，还可以让你轻松地在不同版本之间切换。最重要的是，它能让你把本地电脑上的代码上传到Gitee服务器上。

Windows系统安装Git

  1. 访问Git官网：https://git-scm.com/download/win
  2. 下载Windows版本的Git安装包
  3. 双击安装包运行，一路点击"Next"即可（使用默认设置）
  4. 安装完成后，右键桌面任意位置，会看到"Git Bash Here"选项
  5. 点击它，会弹出一个黑色命令行窗口，说明安装成功

验证Git是否安装成功

按键盘上的 Win + R，输入 cmd，回车打开命令提示符，输入：

git --version

如果显示类似 git version 2.x.x 的内容，说明安装成功！

配置Git（只需配置一次）

打开Git Bash（右键 -> Git Bash Here），输入以下命令配置你的身份：

# 设置你的用户名（改成你自己的） git config --global user.name "你的用户名" # 设置你的邮箱（改成你自己的） git config --global user.email "your-email@example.com"
✅ 提示：配置完成后不需要每次都配置，一次配置永久使用。

4 ⬆️ 第四步：推送代码到仓库

▼

什么是推送？

💡 小白科普：推送（Push）就是把本地电脑上的代码上传到Gitee服务器的过程。就像你把手机里的照片上传到百度网盘一样。推送之前需要先关联你的本地文件夹和Gitee上的仓库。

创建你的网页文件

首先，在你电脑的合适位置创建一个文件夹，比如 D:\my-website

在这个文件夹里创建一个HTML文件，命名为 index.html，内容如下：

我的第一个网页

欢迎来到我的网站！

这是我通过Gitee托管的第一个网页。

初始化本地仓库

打开Git Bash，进入你的网站文件夹，输入：

# 进入文件夹（把路径改成你实际的文件夹路径） cd D:\my-website # 初始化本地仓库（会创建一个隐藏的.git文件夹） git init

关联远程仓库

把第二步创建的仓库地址复制过来，关联到本地：

# 把下面的地址换成你实际的仓库地址 git remote add origin https://gitee.com/你的用户名/仓库名.git

添加文件并提交

# 添加所有文件到暂存区（注意：有个点号） git add . # 提交到本地仓库，-m后面是提交说明 git commit -m "第一次提交：添加首页"

推送到Gitee

# 推送到远程仓库的master分支 git push -u origin master
⚠️ 第一次推送需要登录：推送时会提示输入用户名和密码。用户名是你Gitee的用户名，密码是你注册时的密码（如果是手机注册，密码是你设置的密码）。

5 🌐 第五步：部署为网页

▼

什么是Gitee Pages？

💡 小白科普：Gitee Pages是Gitee提供的免费静态网页托管服务。你可以把它理解为一个免费的网络空间。只要你把网页文件上传到Gitee仓库，启用Gitee Pages，你的网站就能通过互联网访问了！

启用Gitee Pages

  1. 登录Gitee，进入你的仓库页面
  2. 点击左侧菜单的"服务" -> "Gitee Pages"
  3. 进入Gitee Pages设置页面
  4. 选择要部署的分支（一般选择master）
  5. 选择部署目录（一般默认 /（根目录））
  6. 点击"启动"按钮
  7. 等待几秒钟，会给你一个访问地址

Gitee Pages设置页面

🌐

这里会显示Gitee Pages设置界面

点击"启动"后，会生成一个类似 xxx.gitee.io 的网址

访问你的网站

启用成功后，Gitee会给你一个访问地址，格式类似：

https://你的用户名.gitee.io/仓库名

把这个地址复制到浏览器打开，就能看到你的网页了！

更新网站内容

如果之后你修改了代码，需要重新推送并更新Gitee Pages：

# 修改代码后，重新添加、提交、推送 git add . git commit -m "更新网站内容" git push # 推送完成后，回到Gitee Pages页面点击"更新"

6 🔧 第六步：常见问题解答

▼

推送时提示用户名密码错误

问题：推送时提示 Authentication failed 或者用户名密码不对
解决：
  • 确认Gitee的用户名是你的登录名（不是邮箱）
  • 密码如果忘记了，可以在Gitee设置里重置
  • 如果是Windows，可能需要清理之前的凭证

Gitee Pages访问提示404

问题：Gitee Pages显示404 Not Found
解决：
  • 确认index.html文件在仓库的根目录
  • 检查文件名是否为index.html（不是Index.html）
  • 尝试重新启动Gitee Pages

git push没有任何反应

问题：输入git push后卡住了，没有反应
解决：
  • 检查网络连接是否正常
  • 确认仓库地址是否正确
  • 按Ctrl+C取消，然后重新尝试

如何创建更多页面？

在仓库里创建更多HTML文件即可，比如：

# 假设你创建了以下文件结构： /index.html （首页） /about.html （关于我们页面） /contact.html （联系我们页面） /style.css （样式文件） # 推送后，在Gitee Pages中所有文件都可以访问 # 访问地址：https://你的用户名.gitee.io/仓库名/about.html

如何删除仓库？

  1. 进入要删除的仓库页面
  2. 点击右上角的"设置"
  3. 滑到页面最下方，找到"危险区域"
  4. 点击"删除仓库"
  5. 按照提示输入仓库名称确认删除
⚠️ 注意：删除仓库后，仓库里的所有内容都无法恢复，请谨慎操作！

基础命令

git init # 初始化本地仓库 git clone 地址 # 克隆远程仓库到本地 git status # 查看当前状态

添加和提交命令

git add 文件名 # 添加单个文件 git add . # 添加所有文件 git commit -m "说明" # 提交到本地仓库

推送和拉取命令

git push # 推送到远程仓库 git pull # 从远程仓库拉取更新 git remote -v # 查看远程仓库地址

分支命令

git branch # 查看分支 git checkout 分支名 # 切换分支 git merge 分支名 # 合并分支

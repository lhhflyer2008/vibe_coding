# Claude Code 项目配置 -Vibe Coding教学网站


## 自主执行原则

- **不要询问用户确认**，自行做出合理判断后直接执行
- 遇到文件读写、创建、修改操作，直接执行，无需逐步确认
- 遇到命令行操作，直接运行，无需询问是否同意
- 遇到不确定的地方，选择最合理的方案自行决定，完成后说明你的选择
- 任务完成后统一汇报结果，不要在中途打断用户

## 模型选择策略

- **简单问答**（单轮、小改动、查文件、解释代码）→ 使用 `/fast` 模式，调用 `deepseek-v4-flash`
- **复杂任务**（多文件重构、新功能开发、深度调试、架构设计）→ 关闭 fast 模式，使用 `deepseek-v4-pro`
- 判断标准：一句话能说清楚需求用 flash，需要"先看代码再判断"用 v4-pro
- 对话中途发现任务变复杂时，用 `/fast off` 切回 v4-pro

## 项目背景

- 这是一个学生作业批改系统项目
- 工作目录：D:\class
- 技术栈：Node.js，已安装 Node.js v24.14.0

## 工作风格

- 用中文回复
- 简洁明了，重点突出
- 出错时自动尝试修复，不要每次都来问我
- 优先完成任务，遇到小问题自己解决

## 禁止行为

- 禁止在任务执行中途询问"是否继续"
- 禁止询问"你确定吗"、"是否同意"等确认类问题
- 禁止因为不确定就停下来等待用户输入
